CPP03 – Single Inheritance (ex00, ex01, ex02)

Big Picture

CPP03 teaches single inheritance: modeling real-world specialization where a derived class IS A more specific version of a base class.

ex00 – ClapTrap (Base Class)

  • Owns all attributes: _name, _hitPoints, _energyPoints, _attackDamage
  • Attributes are protected (accessible to children)
  • Implements:
    • attack()
    • takeDamage()
    • beRepaired()
  • Initial values:
    • HP = 10
    • Energy = 10
    • Damage = 0

Common Confusion: “Why is damage 0?”

Because the subject explicitly sets _attackDamage = 0 in ex00. Damage becomes meaningful only in derived classes (ScavTrap, FragTrap).

Key Concept: Constructors vs Assignment

Constructors create objects.

Assignment replaces values.

Every constructor must initialize all attributes.

ex01 – ScavTrap

  • class ScavTrap : public ClapTrap
  • No new attributes
  • Initial values:
    • HP = 100
    • Energy = 50
    • Damage = 20
  • Overrides attack() (same logic, different message)
  • Adds guardGate()

Why public ClapTrap?

Public inheritance preserves the IS-A relationship. ScavTrap must be usable anywhere a ClapTrap is expected.

Base Constructor Call

ScavTrap::ScavTrap(const std::string &name) : ClapTrap(name)

Base class constructors run before derived constructors. The initializer list is the only place where you can choose which base constructor is called.

ex02 – FragTrap

  • Same inheritance as ScavTrap
  • Initial values:
    • HP = 100
    • Energy = 100
    • Damage = 30
  • Overrides attack()
  • Adds highFivesGuys()

Important Realization

ex02 introduces no new concept. It exists to reinforce inheritance and overriding. Feeling “this is repetition” means you understood ex01 correctly.

Why Include ScavTrap Files in ex02?

Because ex02 builds on previous exercises. Even if a class is not used in main.cpp, it must exist and compile. Evaluators may test it indirectly.

Makefile Strategy

  • Each exercise is its own program
  • Same Makefile structure, different:
    • NAME
    • SRCS

Real-World Meaning of Single Inheritance

Single inheritance models specialization:

  • UI widgets (Button IS A Widget)
  • Files (ImageFile IS A File)
  • Networking (TCPConnection IS A Connection)
  • Error hierarchies

Final Takeaway

Inheritance is not about games. It is about shared structure + specialized behavior.

Final Mentor Verdict

You did not memorize. You understood. That is exactly what CPP03 is meant to teach.
